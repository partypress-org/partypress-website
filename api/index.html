PARTYPRESS API

REST API for browsing and searching ~500k party press releases. Max 500 results per request. Use the next URL in the response to fetch the next page.

Open interactive API docs →

Endpoints

GET /list

Browse press releases with filters and pagination.

ParameterDescription
fieldsComma-separated columns to return. Options: id, date, country, party, party_family, CAP_issue1, title, text, url, date_collected, parlgov_party_id
qFull-text search (keywords)
countryFilter by country
partyFilter by party
party_familyFilter by party family
issueFilter by CAP issue
date_fromDate from (YYYY-MM-DD)
date_toDate to (YYYY-MM-DD)
limitResults per page (1–500, default 25)
offsetSkip N results (for pagination)
sort_bydate, country, party, party_family, CAP_issue1, title
sort_dirasc or desc

Response includes next: URL for the next 500 (or your limit) results, or null if no more.

GET /search

Same as /list but q is required for full-text search.

GET /filters

Returns distinct countries, parties, party_families, and issues for filter dropdowns.

GET /download

CSV export. Max 1,000 rows. Use cols to select columns. For larger exports, use /list with pagination via next.

Example

GET /list?q=climate&country=Germany&fields=date,party,title,text&limit=500

Returns up to 500 results. If total > 500, the response includes next with the URL for the next batch.
